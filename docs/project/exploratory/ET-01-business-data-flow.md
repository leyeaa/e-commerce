# ET-01: business and data-flow session

## Mission

Evaluate the authenticated cart-to-order journey and trace the transformation of
product, quantity, price, user, and checkout data through the React interface,
API responses, Django models, and PostgreSQL records.

## Charter

Explore the normal shopping journey using the Guidebook and Fed-Ex tours to
discover inconsistencies between documented behaviour, displayed values, API
responses, and persistent database state.

## Timebox

90 minutes.

## Suggested coverage

- Login and authenticated session.
- Add one and multiple products.
- Increase and decrease quantities.
- Remove and re-add a product.
- Compare UI cart total with API cart total.
- Complete checkout with both payment-method choices.
- Inspect order, order items, prices, quantities, user, and cart cleanup.
- Refresh and navigate backward after checkout.
- Repeat an order submission.

## Evidence

Capture relevant UI screens, HTTP requests and responses, and before-and-after
database state. Complete session-template.md during execution.
